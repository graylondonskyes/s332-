# 08 — AFTER-DIRECTIVE OPTIONAL EXPANSION

This file does **not** replace the directive.
This file only defines when the old `extra-shit` folder becomes allowed.

## Rule
An optional expansion is allowed only after the directive lane it depends on is already grounded.

## Gating table
- route-pack enhancements: allowed only after Batch 1 is complete
- richer proof / customer-facing service-doc polish: allowed only after Batch 2 is complete
- scan-first field workflows: allowed only after Batch 3 is complete
- richer media-note workflows: allowed only after Batch 4 is complete
- predictive prioritization / smarter route suggestions: allowed only after Batch 5 is complete
- deeper board / operator command visuals: allowed only after Batch 6 is complete
- trip-level premium ops layers: allowed only after Batch 7 is complete
- hybrid online enrichments: allowed only after Batch 8 and the offline proof gates are complete

## What this means in practice
The old `extra-shit` folder is now demoted to **phase-after** material.
It is not the source of truth.
It is not the starting point.
It is not allowed to hijack the directive-first build.
