# SOLE / SkyeSol Visual Standard

This is the baseline design floor for SkyeWebCreatorMax output.

Source calibration:
- `SOLEnterprises.org/SOLEnterprises.org-main/Pages/ObsidianNoir.html`
- `SkyeSol/skyesol-main/Bundles/assets/site.css`
- `SkyeSol/skyesol-main/css/style.css`
- `SkyeSol/skyesol-main/SkyeDocx/index.html`
- `AbovetheSkye-Platforms/SkyeForgeMax/assets/forge.css`

Required traits:
- cinematic dark base with layered radial gradients, aurora/noise overlays, and glass panels
- gold/violet/blue, obsidian/gold/magenta/mint, or magma/gold founder palettes
- sticky operating nav with small uppercase labels and high-intent CTA buttons
- longform page architecture: hero, side stack, proof grids, console/portal area, process, offer tiers, intake
- dense but controlled information, not generic empty marketing sections
- buttons and links must work; every hash link must target a real section
- forms must provide local status feedback until backend endpoints are connected
- generated artifacts must include a source/provenance note pointing to donor and house-style references
- interactive canvas/3D is a supporting cinematic layer, not the whole product
- generated websites must satisfy `design-vault/library/house-style/sole-skye-section-corpus.json`
- imported donor repos are reference libraries; they do not become generation-approved until their output is upgraded to this house standard

Disallowed:
- one-screen toy pages
- fake dashboards with dead buttons
- anchors without `href`
- generic neon SaaS styling that ignores the house palette and page density
- cards-only layouts with no operating flow, proof architecture, or conversion path
