What this number means

This is the hard-nosed present-value read on the codebase itself. It prices the SkyeHands repository as a real integrated operator platform — not a point feature, not a single SaaS product, and not a prototype. The $86.8M code-floor is what the platform is worth as existing code at the time of appraisal. The raise posture and strategic upside ranges exist separately because category creators are funded and acquired on what the platform can become once the code lead converts into commercial proof. The remaining completion work is now mostly live provider configuration, production persistence, and deployment execution — no design unknowns remain.

Why the floor is this high

The SkyeHands platform is not a point feature. It unifies a signed event bus, a 13-brain specialized agent mesh, a governed full-stack AI routing gateway, a 9-app productivity suite, multi-tenant SaaS infrastructure, a 90GB AI brain drive, 11 offline-first PWAs, a sports platform, a gaming engine, an email client, a document platform, and a proof-discipline governance system into one operator-owned surface. No single competitor ships all seven layers together. A strategic buyer is not pricing a mockup — they are pricing a real architecture, a real build lead, and a real category position. The code inventory represents 8–12 years of 4–6 engineer output at market rates.

Boundary conditions

  • The $86,800,000 code-floor is intentionally hard-nosed and present tense. It is based on what is currently in the repository — not what it will generate in revenue.
  • Projected revenue belongs in the upside case, not in the code-floor number.
  • The raise-posture range ($115M–$190M pre-money) exists because category creators are funded on what the platform can become once the code lead converts into commercial proof.
  • The strategic-upside range ($600M–$3.2B) is named to make the boundary between present value and future acquisition value explicit — not to inflate the current floor.
