# 25 — POST-DIRECTIVE INTERACTIVE WALKTHROUGH CANON

This pass adds a product-education layer that is built into the shipped app instead of depending on external explanation.

## Locked standards
- Walkthroughs must change screens while they run.
- Settings must contain a tutorial center.
- The app must provide multiple walkthroughs that cover the major operational lanes.
- Progress must be remembered locally.
- The walkthrough system should make the product feel transparent, teachable, and easier to adopt.

## Landed surfaces
### Routex
- Topbar Tours button
- Dashboard walkthrough launchpad
- Settings walkthrough center card
- Guided walkthrough overlay with progress + step navigation
- Guided tours for:
  - core navigation
  - route ops
  - operator transparency stack
  - security / recovery

### AE FLOW
- Topbar Tours button
- Settings walkthrough center card
- Guided walkthrough overlay
- Guided tours for:
  - core usage
  - Routex bridge
  - settings / backup / trust
