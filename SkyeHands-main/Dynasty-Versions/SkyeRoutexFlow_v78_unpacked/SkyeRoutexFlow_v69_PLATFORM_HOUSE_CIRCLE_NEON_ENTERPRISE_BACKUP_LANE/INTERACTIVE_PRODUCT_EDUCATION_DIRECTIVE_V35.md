# INTERACTIVE PRODUCT EDUCATION DIRECTIVE — V35

## Intent
The app must explain itself in-product.
Walkthroughs are not passive notes.
They must move the user through the actual screens, tabs, controls, and live operating lanes they need to use.

## Required behavior
- Build guided walkthroughs that change screens while the walkthrough is running.
- Put a tutorial center inside Settings so the help system is permanently accessible.
- Cover the primary product lanes instead of only onboarding the first screen.
- Track walkthrough progress locally so operators can resume or replay training.
- Keep the product transparent enough that a new user can finish the walkthroughs and understand how to operate the app without outside explanation.

## What V35 lands
### Routex
- topbar Tours entry point
- dashboard walkthrough launchpad
- settings tutorial center card
- four guided walkthrough sets:
  - Start here
  - Route ops
  - Operator stack
  - Security and recovery

### AE FLOW
- topbar Tours entry point
- settings tutorial center card
- three guided walkthrough sets:
  - Core app tour
  - Routex bridge
  - Settings, backup, and trust

## Killer-app usability upgrades in this pass
- mission-based tour launcher surfaces
- progress tracking for completed walkthroughs
- step-by-step screen switching instead of detached instructions
- in-product education embedded directly into the command surfaces
