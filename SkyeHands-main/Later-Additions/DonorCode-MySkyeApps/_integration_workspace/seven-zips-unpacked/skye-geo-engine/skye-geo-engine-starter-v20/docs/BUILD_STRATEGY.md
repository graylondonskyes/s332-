# Skye GEO Engine — Build Strategy

This starter is designed to attack the same public product surface marketed by BabyLoveGrowth: automated content production, AI-search visibility monitoring, technical GEO auditing, CMS publishing, and a network-driven growth loop. The public site claims optimized content generation, backlink distribution, AI visibility tracking, technical auditing, Reddit visibility support, multilingual content, CMS integrations, and agency resale options. This starter tackles the technical-audit, planning, payload-mapping, and visibility-testing lanes first, because they are measurable and can be proven without external provider variables. The backlink-network and automated publishing lanes are the heavier moat layers and should be added after the foundation is hardened.

## What already exists in this starter

- Real Cloudflare Worker API.
- Real UI at `/app` with no dead controls.
- Real live-page technical audit endpoint.
- Real 30-day content-plan generator.
- Real LLM prompt-pack generator.
- Real CMS payload mapper for WordPress, Webflow, Shopify, and generic API use.

## What beats the competitor

To beat the BabyLoveGrowth class of tools, do not stop at content generation. Win on proof, control, and system depth.

1. Build a better technical audit than theirs. Expand beyond metadata into crawl depth, duplicate intent, entity extraction, FAQ coverage, and answer-engine readiness scoring.
2. Make prompt visibility replayable. Every prompt run should store provider, timestamp, output, mention share, citation share, and competitor overlap.
3. Add a citation ledger. Every generated article should retain source provenance, retrieval timestamps, and claim-to-source mapping.
4. Add deterministic publishing logs. Every post operation should store request, response, destination record, and reconciliation state.
5. Build an agency console that actually scales. Multi-tenant, white-label exports, client workspaces, role controls, seat caps, and usage metering.
6. Build a stronger network moat. The backlink/distribution engine is the true defensibility layer.
