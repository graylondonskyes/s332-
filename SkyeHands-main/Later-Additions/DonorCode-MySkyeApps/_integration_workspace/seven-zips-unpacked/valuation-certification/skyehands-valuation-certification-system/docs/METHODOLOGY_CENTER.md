# Methodology Center

## Valuation families

### Build Floor Value
Replacement-cost oriented baseline using file count, code line count, frameworks, services, descriptors, documentation, CI surfaces, test evidence, runtime proof, repair intelligence, and trust-pack generation.

### Functional Certified Value
Current evidence-backed operational value using route signals, launch profiles, runtime proof checks, smoke/test presence, unresolved issue pressure, repair readiness, and trust posture.

### Enterprise Uplift Value
Value created by SAML, SCIM, RBAC, audit, export, admin, tenant, deploy, policy-linked signals, and public-trust issuance readiness.

### Integration Value
Provider-weighted value ledger for 2026. Each provider receives depth scoring from keyword evidence, manifest presence, descriptor presence, env-var evidence, and file spread. The ledger is in `config/integration_ledger_2026.json`.

### IP / Strategic Uplift Value
Signals for custom branded logic, AI depth, workflow depth, proprietary modules, repair subsystem prompt-pack structure, and non-commodity orchestration.

### Operational Burden Value
Credits the complexity of shipping real multi-language, multi-provider, multi-runtime, trust-ready systems before traction exists.

### Traction Uplift Value
Reserved lane. Weight remains intentionally low until live traction data is supplied.

## Confidence policy

- **High**: runtime proof passed on at least two checks, README exists, and tests are present; or execution is fully proven; or repair + trust lanes are both smoke-backed.
- **Medium**: at least one runtime check passed.
- **Low**: advisory-only scan with weak runtime proof.

## Certification policy

- **Unverified / Incomplete**: score below twenty.
- **Build Present**: score twenty or above.
- **Functional**: score forty or above.
- **Release-Ready**: score sixty or above.
- **Enterprise-Ready**: score seventy-five or above.
- **Sovereign / Platform Grade**: score eighty-eight or above.

## Portfolio certification

Portfolio score aggregates workspace-level issued value, overall score, confidence posture, patch adoption, enterprise-ready density, issue pressure, repair-lane readiness, and trust posture. The resulting portfolio certification uses the same level ladder as individual workspaces.

## Authority, repair, and trust

Authority artifacts are locally signed certification outputs that cover core valuation artifacts and can be re-verified later. Repair intelligence emits ranked remediation plans and a provider-ready prompt envelope backed by an offline fallback. Public trust readiness emits a verified local chain, CSR, ACME challenge files, and issuance helper surfaces.
