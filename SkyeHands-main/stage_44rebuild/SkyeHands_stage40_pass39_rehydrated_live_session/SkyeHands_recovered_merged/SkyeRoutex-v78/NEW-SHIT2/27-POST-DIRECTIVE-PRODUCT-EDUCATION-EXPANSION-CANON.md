# 27 — POST-DIRECTIVE PRODUCT EDUCATION EXPANSION CANON

This canon locks the v36 post-directive expansion:
the app should not merely contain walkthroughs, it should function like a self-teaching product.

## Locked principles
1. walkthroughs must move through real screens and live controls
2. first-run users should be offered guided onboarding inside the product
3. deep surfaces should expose contextual "Guide" launchers instead of forcing users back to docs
4. the product should show a coverage matrix proving what has guided education and what still needs to be run
5. Routex and AE FLOW should both follow this standard

## Routex v36 additions
- 8 dedicated walkthroughs
- recommended onboarding sequence
- dashboard launchpad + settings tutorial center card
- contextual guide launchers on major deep-surface controls
- coverage matrix inside the tutorial center

## AE FLOW v36 additions
- 6 dedicated walkthroughs
- recommended onboarding sequence
- intake/settings launch surfaces
- contextual guide launchers on major bridge controls
- coverage matrix inside the tutorial center

## Honest boundary
This canon covers shipped in-app education structure.
It is not a claim that every v36 walkthrough interaction survived a full unrestricted browser smoke in this sandbox.
