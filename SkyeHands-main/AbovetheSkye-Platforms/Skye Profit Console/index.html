Core Modules

Six Modules. One Console.

These are the hosted app modules this shell points to. The modules themselves are not implemented in this local folder.

📊

Dashboard

Live snapshot of Revenue, Expenses, Net Profit, Net Margin, Unreconciled Deposits, and Audit Readiness score for the active month. Quick-action buttons to jump to any module.

💰

Revenue Log

Log every sale with client name, category, payment method, linked invoice number, and notes. Export to CSV for your accountant or auditor in one click.

📋

Expense Tracker

Track expenses by COGS vs OpEx — separating delivery costs from overhead so your margin story holds up under scrutiny. Vendor, category, method, all logged.

🧾

Invoice Manager

Create invoices with client, issue date, due date, and amount. Track status (draft → paid → overdue) and log payment references so deposits can be matched.

🏦

Bank Deposits

Every dollar that hits your account gets its own deposit record. Mark each as reconciled once matched to revenue. Unreconciled count surfaces on the dashboard immediately.

🔄

Reconciliation

Match bank deposits to revenue entries — the last step that closes your monthly books. One-click reconcile with an audit trail of what matched to what.

Offline-First Architecture

Your Data Stays With You

The deployed SkyeProfitConsole runtime is described as browser-local. This folder now includes its own local runtime and local browser storage, but it still does not prove the hosted sync engine.

📱

Hosted App Works Offline

The broader hosted runtime still owns the richer offline experience. This local folder now stores a browser-local ledger snapshot and can persist it through the in-folder runtime, but it does not prove the hosted sync or installed-app path.

📤

Hosted JSON Backup

JSON backup and import are deeper in the deployed app. This folder proves local JSON export and a local runtime-backed snapshot, not hosted import workflows.

Hosted CSV Export
